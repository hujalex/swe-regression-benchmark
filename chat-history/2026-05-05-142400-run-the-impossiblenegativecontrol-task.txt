‚Źļ Found the root cause. Verifiers' interleaved scheduler runs each rollout as
  run_one(i) async tasks. The generation step is gated by max_concurrent=1, but
  scoring is a separate await ‚ÄĒ so the order can be:

  1. Rollout 1: setup_state (git reset) ‚Üí agent edits ‚Üí generation done ‚Üí
  releases semaphore
  2. Rollout 2: setup_state (git reset, wipes rollout 1's edits) ‚Üí agent edits
  3. Rollout 1: score_rollout runs ‚Üí sees rollout 2's workspace, empty diff vs
  baseline (or diff of rollout 2!)

  NUM_EXAMPLES=1 worked because there's no rollout 2 to clobber it.

  The fix: do all the scoring inside the rollout itself, before yielding
  control. Override rollout() to call the parent's logic, then immediately
  snapshot diff + run tests, stashing the result into state. reward_fn then just
   reads from state.
